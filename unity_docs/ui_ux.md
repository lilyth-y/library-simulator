# UI/UX Specification

## 1) UI 원칙
- 정보는 간결하게, 색상은 파스텔 톤 유지
- 작은 화면에서도 읽기 쉬운 큰 타이포
- 실패/경고는 부드러운 피드백으로 전달
- 복잡도 숨김: 내부 수치/확률은 비공개, 상태 등급으로 표현

## 2) 주요 화면 구성

### 2.1 메인 메뉴
- 이어하기 / 새 게임 / 설정
- 사운드, 언어, 그래픽 옵션
- 모드 선택: 기본(상한 7단계) / 도전(상한 해제)

### 2.2 인게임 HUD
- 상단: 현재 시간, 만족도(평판), 코인, Day 표시
- 상단: 하루 진행 바 + 페이즈 아이콘(일반/러시)
- 상단 또는 우측: 혼잡도(Chaos) 등급 게이지(숫자 숨김)
- 좌측: 업무 목록(퀘스트)
- 우측: 반납 큐 상태, 경고 아이콘
 - 이벤트 발생 시 경고 배너(돌발 러시/붕괴 등)

### 2.3 도서관 PC 화면
- 검색창
- 결과 리스트(카테고리/상태 표시)
- 선택 시 해당 선반 강조 표시

### 2.4 책 정보 팝업
- 제목, 상태, 카테고리
- 수리 가능 여부 표시

---

## 3) 입력 흐름

### PC
- 클릭: 대상 선택/집기
- 드래그: 들고 이동
- 우클릭: 놓기/취소
- 단축키: PC 열기, 지도 토글

### 모바일
- 탭: 선택/집기
- 드래그: 이동
- 길게 탭: 정보 보기
- 화면 가장자리 버튼: PC, 지도, 퀘스트

---

## 4) 피드백 규칙
- 올바른 배치: 부드러운 "딸깍" 사운드 + 파티클
- 잘못된 배치: 가벼운 흔들림 + 노란 경고 테두리
- 붕괴: 먼지 파티클 + 짧은 경고 사운드
- 돌발 러시: 상단 배너 + 짧은 경고음 + UI 색상 강조
- 난이도 단계 상승: 작은 토스트 + UI 강조(과도한 경고는 금지)

---

## 5) 접근성
- 큰 글자 모드(옵션)
- 색약 모드(색상 대비 강화)

---

## 6) 정보 노출/숨김 정책

### 6.1 항상 보여야 할 것
- 현재 시간/하루 진행 바
- 업무 목록(현재 해야 할 일)
- 혼잡도 등급(색상/아이콘)
- 반납 큐 상태
- 즉시 위험 경고(이벤트 배너)

### 6.2 상황에 따라 보여야 할 것
- 책 정보(제목, 상태, 카테고리)
- 수리/검색 결과 UI
- 오늘 정산/업그레이드 선택
- 손님 상태(소란/진정) 간단 아이콘

### 6.3 숨길 것(복잡도 비공개)
- 난이도 단계/증가 수식
- 사건 확률/쿨다운 수치
- 내부 혼잡도 수치 및 임계치
- AI 상태머신 세부 상태
- 물리 안정성 계산 파라미터

### 6.4 현실감 유지 원칙
- 플레이어는 "느껴지는 분위기"만 인지하도록 설계
- 문제의 원인은 아이콘/톤으로 전달하되 수치화는 지양

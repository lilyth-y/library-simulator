# Game Design Brief

## 1) 하이 컨셉
두근두근타운 감성의 아기자기한 도서관에서
플레이어가 사서가 되어 책을 정리하고,
손님을 응대하며, 도서관을 성장시키는
스타일라이즈 경영 시뮬레이션.

## 2) 핵심 기둥(Pillars)
1. 따뜻한 일상 경험
   - 정돈, 도움, 성취의 루프
2. 손맛 있는 물리 상호작용
   - 집기, 놓기, 쌓기, 넘어짐
3. 명확한 경영 지표
   - 평판, 수익, 만족도, 업무량
4. 작은 성공의 반복
   - 짧은 목표 달성과 보상

## 3) 타깃 플랫폼/입력
- PC + 모바일
- PC: 키보드/마우스
- 모바일: 터치/드래그/탭, 간단한 제스처

## 4) 카메라/컨트롤 기본안
- 카메라: 3D 상단 시점(약 35~45도), 줌/회전 제한
- 이동: 터치 조이스틱(모바일), WASD(PC)
- 상호작용: 탭/클릭로 집기, 드래그로 놓기

## 5) 코어 루프
1. 출근/오픈
2. 손님 방문 -> 요청/반납/질문
3. 책을 찾아 정리/배치/수리
4. 보상 획득(평판/코인)
5. 하루 마감 -> 업그레이드/확장

## 6) 톤/긴장감 정책
- 힐링 톤을 유지하되, 약간의 긴장감을 상시 유지
- 혼잡도(Chaos)가 높아질수록 업무 속도/사건 확률 증가
- 일반 페이스에서도 돌발 러시(20~30초)가 가끔 발생 가능
- 돌발 러시는 하루 0~1회, 연속 발생 금지(쿨다운 적용)
- 상태 누적이 높으면 사건이 연쇄로 이어질 수 있음(하루 최대 1회)

## 7) 난이도 곡선(매운맛 A)
- 진행일(dayIndex)이 증가할수록 완만하게 난이도 상승
- 난이도 단계는 3일마다 +1(예: 1~3일: 1단계)
- 단계 상승 효과(상한 존재):
  - 업무 생성 간격 5% 감소
  - 소란/실수 발생 확률 5% 증가
  - 돌발 러시 확률 5% 증가
- 매운맛 A 기준:
  - 실수 1~2회는 회복 가능하도록 설계
  - 정리/수리 완료 시 혼잡도 감소 폭을 크게 유지
  - 하루 정산 시 혼잡도 부분 리셋

## 8) 세션 길이/하루 구조 (5분 기준)
- 하루 = 300초
- 0:00~0:30 오픈(완화 구간, 튜토리얼 성격)
- 0:30~3:00 일반(기본 업무 페이스)
- 3:00~4:30 러시(업무 증가 + 사건 1회)
- 4:30~5:00 정리/정산(신규 업무 중단)

## 9) 진행/성장 요소
- 도서관 확장(구역, 서가 종류)
- 시설 업그레이드(반납함, 수리대, 검색 PC)
- 직원 고용(자동 정리, 고객 응대)
- 장식/코스메틱(아기자기한 마을 감성 강조)

## 10) 실패/압박 요소
- 반납 큐 초과 -> 혼란 상태
- 서가 붕괴 -> 패널티/정리 시간 증가
- 소란 손님 -> 만족도 하락

## 11) MVP 범위(권장)
- 맵 1개, 기본 구역 3개(입구/열람/서가)
- 책 3~5종 카테고리, 20~40권
- 손님 2종(일반/소란)
- 상호작용 3종(집기/놓기/수리)
- 기본 UI(업무 목록, 만족도, 시간)

## 12) 범위 외(초기 제외)
- 대규모 타운 탐험
- 멀티플레이
- 복잡한 경제 시뮬레이션
